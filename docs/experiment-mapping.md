# AniVerseX Experiment Mapping

## Mini Project Title

AniVerseX: Anime Recommendations and Reviews

## Requirement to Module Mapping

### 1. Client-side form design and validation

- File: `frontend/html-js/registration.html`
- Features:
  - registration form
  - input validation
  - password confirmation
  - error messages

### 2. GET and POST

- File: `frontend/jquery-ajax/index.html`
- Backend: `backend-node/server.js`
- Features:
  - live GET anime list from API
  - live POST new anime recommendation to API

### 3. JavaScript events and objects

- File: `frontend/html-js/index.html`
- Script: `frontend/html-js/script.js`
- Features:
  - `click`, `mouseover`, `change`, `submit`
  - class-based object model
  - review objects and in-memory updates

### 4. jQuery and AJAX

- File: `frontend/jquery-ajax/index.html`
- Script: `frontend/jquery-ajax/app.js`
- Features:
  - jQuery DOM manipulation
  - AJAX form submit
  - dynamic list rendering

### 5. React API

- File: `frontend/react-app/index.html`
- Script: `frontend/react-app/app.js`
- Features:
  - component rendering
  - list display using `map`
  - form interaction using React state

### 6. Spring Boot MVC + CRUD + MySQL

- Folder: `backend-springboot/`
- Features:
  - entity, repository, controller
  - JSP pages
  - create, list, edit, delete flow
  - MySQL schema script

### 7. Node.js and Express multi-tier application

- Folder: `backend-node/`
- Features:
  - Express server
  - routes
  - MongoDB model
  - REST endpoints

### 8. XML DOM processing

- Folder: `xml-module/`
- Files:
  - `anime.xml`
  - `viewer.html`
- Features:
  - parse XML using DOMParser
  - render anime details in browser

## Presentation Summary

AniVerseX combines frontend validation, event handling, AJAX, React, backend CRUD, and XML parsing in a single anime-based mini project. Each module corresponds directly to one or more syllabus experiments.
